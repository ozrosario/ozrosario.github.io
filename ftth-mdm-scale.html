Technical Architecture

VETRO FiberMap API — Confirmed Capabilities

VETRO operates a RESTful API built on OpenAPI 3.0. Authentication via token (generated in Developer Portal → Tools → Tokens). Full docs gated to customers — request API token + permissions from the operator IT on Day 1.

API Capability Endpoint Use in Market Scoring
Address Match & Serviceability /v2/address/match/{address} Confirm whether a target parcel falls inside an existing plan, build zone, or unserved area
Geometry Proximity Query /api/layer/geometriesNear/{lng}/{lat} Pull existing conduit, backbone, poles, and nodes near a candidate market zone — drives build feasibility score
Layer Feature Read/Write GET/POST/UPSERT features Inject scored market zones as custom overlay layers visible in the live the operator VETRO instance
Plan & Zone Sync Plans = towns · Sub-plans = neighborhoods Tie scoring model outputs to active build plans; trigger alerts when a zone crosses priority threshold
GeoJSON Feature Upsert POST features with layer_id + plan_id Push AI-scored heatmap polygons directly into VETRO as a custom scoring layer — visible to full engineering and build team
⚠️
Critical Day 1 Action

Full VETRO API access requires a customer API token from the operator's live instance. Token generation lives at VETRO Dashboard → Developer Tools → Tokens → Assign Permissions. Surfaced at engagement kickoff, not mid-build. IT approval cycles can delay Phase 1 by 2–3 weeks.

Phase 1 Workflow — 30–60 Days

✅
No Budget Required for Phase 1

This entire workflow runs on public APIs and tools already in operation. Phase 1 proves the model with zero incremental spend — making it safe to present to the CFO before the Phase 2 SOW is approved.

1

Define Target Geography

Map existing active plans in VETRO — the lead county live, a major metro and a queued metro County queued. Establish geographic perimeter of Tier 1 priority zones.

VETRO FiberMap — existing the operator instance → Geographic perimeter map of priority zones
2

Pull FCC Fabric — Competitive Gap Analysis

Query FCC National Broadband Map API by census block. Identify unserved and underserved areas within target geography.

FCC API — free public endpoint → Underserved census block heatmap
3

Layer Census ACS Data

Pull housing density, renter/owner ratio, and median income by census tract. Calculate revenue potential score: households × addressable ARPU ceiling.

US Census API — ACS 5-year estimates → Revenue potential score per tract
4

Map HOA/COA Concentration

Pull CA Secretary of State entity data for HOA/COA density by zip code. Layer over LoopNet/CoStar parcel data to confirm unit counts. This is the operator's core market moat.

CA SOS registry + LoopNet/CoStar → HOA density heat layer by zip
5

Query VETRO geometriesNear API

For each candidate zone centroid, query existing infrastructure proximity: conduit, backbone, poles, nodes. Drives build feasibility score and cost-to-connect estimate.

VETRO API — /api/layer/geometriesNear/{lng}/{lat} → Build feasibility score by zone
6

Composite Score Calculation

Run the 3-bucket weighted model in Excel/Sheets: 40% Revenue / 35% Build Feasibility / 25% Market Moat. Output Tier 1 / 2 / 3 zone classification with rationale notes per zone.

Excel / Google Sheets — weighted scoring model → Zone Priority Stack Rank v1
7

Push Scored Zones to VETRO Overlay

Use VETRO GeoJSON Feature Upsert API to push scored zones as a custom overlay layer. Color-coded Tier 1 (green) / 2 (amber) / 3 (gray). Visible to the CDO, build team, and field ops in the live instance.

VETRO API — GeoJSON Feature Upsert with custom layer_id → Live heatmap layer in VETRO
8

Package as Investor-Ready Market Brief

Convert zone stack rank into a CDO/CFO/CEO-ready pipeline narrative: Tier 1 cohort, IRR range estimate, capital deployment sequence, 12-month buildout timeline.

Excel + PowerPoint — hockey stick IRR model → Investor pipeline brief v1

The system does not replace human judgment. It eliminates the 80% of manual data aggregation that delays decisions — freeing the NMD team to focus on market validation, municipal relationships, and deal structuring. Every output is decision-grade: sourced, scored, and explainable to a PE investor.

The Trench Once Thesis — CapEx Offset Strategy

🔌
From Discovery

"Let's kill two birds with one stone — if we can pull conduit and voltage wiring to make-ready charging stations, it's a win-win. Infrastructure optimization 101." Validated by the CDO's acknowledgment that HOAs are the operator's strongest segment — the same segment where EV co-deployment has the highest ROI and lowest friction.

  • Red E Charge — EV co-investment MDU partner; joint trench negotiation framework established

  • Autel Energy — OEM hardware partner (Chris Malm); preferred pricing for co-deployed sites
  • AMPPED — 100+ active SoCal EV charging deployments; site host relationships are NMD-ready assets
CapEx Offset Logic

Every HOA that signs a fiber ROE agreement gets an EV charging conversation in the same meeting.

  • Estimated 10–20% reduction in cost-per-passing on HOA builds (to be calibrated with the operator benchmarks)
  • Site host revenue upside is accretive to the HOA — makes fiber adoption easier to close
  • Single trench, dual revenue streams — fiber + EV charging infrastructure

Open Gaps & Required Resolutions

These are the questions this system must answer before it goes to the C-suite. Surfacing them up front is the discipline — not hiding them.

Build cost benchmarks are unknownCritical
Request the operator's actual cost-per-foot and cost-per-passing from current the active build markets builds in the first 30 days. Without this, IRR estimates are directional only. Do not present to the CFO until this number is confirmed.
Penetration rate assumption is unvalidatedCritical
the CDO referenced "a certain demographic profile that hits their numbers." Get the target take rate and ARPU assumption from CDO/CFO before modeling. Do not assume 30–35% and $70–90 ARPU without confirmation — these are industry benchmarks, not the operator actuals.
VETRO API access may require IT approvalDay 1 Flag
Token generation is gated to VETRO Dashboard → Developer Tools → Tokens → Assign Permissions. Needs IT/GIS team coordination. Surfaced at kickoff, before Phase 1 — IT approval cycles can push Phase 1 delivery by 2–3 weeks.
Data licensing cost is unbudgetedCFO Item
CoStar/LoopNet subscription can run $500–2K/month. Needs CFO sign-off or a Phase 1 workaround using public data only. Present this as a line item in the Phase 1 brief — not as a surprise in month 2.
Multifamily unlock is unsolvedSeparate Workstream
the CDO acknowledged the operator has struggled with MDU/multifamily. HOA strength is confirmed. The bulk play (Hotwire failed here after one year) needs a different thesis. Bulk-lite model or managed WiFi angle may be the wedge — needs a dedicated workstream separate from the HOA scoring model.
Google Fiber brownfield risk is unnamedCompetitive Moat
The CDO confirmed all Google Fiber CA activity is greenfield (master-planned communities only). The operator's HOA brownfield approach is differentiated — but this needs to be a named competitive moat in the investor narrative, not just an implied assumption. Name it explicitly in the pipeline brief.
