The Arc

Four movements, one through-line

The exercise started as "rename some files" and became a full method: take real, sensitive deliverables and reduce each to its method and architecture — provable that the work was done, without exposing who it was done for.

MOVEMENT 01

Context & Standards

Split the context payload into voice rules and site content. Set the canonical scope (15 engagements), the email standard, and the delivery discipline: build everything first, commit once. Crucially — agreed the scrub protocol before touching a single file.

MOVEMENT 02

Sanitize & Build

Theme by theme, card by card. Each artifact got a designated source, a narrative frame, and a single pass that scrubbed identity and enriched content at once. Client names to roles, pursuit framing to delivered-work voice, confidentiality stamps to a redaction line.

MOVEMENT 03

Enrich

Where a thin brief undersold the work, we went back to the richer source. A 1,800-line master dashboard rebuilt with its account strategy surgically removed. A 5,200-line pursuit doc mined down to its strategic concept. A coaching-contaminated map replaced with a real four-pilot AIaaS story — built only after the real numbers arrived.

MOVEMENT 04

Capstone & QC

A redesigned index tying all 15 together — then the part that mattered most: every link verified, every page swept for leaks, every file parsed. Nothing shipped on assumption.

The Sanitization Protocol

A consistent line between exposure and credibility

The same rule ran on every file. Strip what identifies a client or a deal; keep what proves the method and earns trust. Redaction stayed intentional and visible — never accidental.

Scrubbed

  • Client names & brands → sector-anchored neutrals
  • Named individuals → role-only
  • Pursuit / interview / account-strategy framing → delivered-work voice
  • Confidentiality stamps → "method & approach showcase" redaction line
  • Leaky filenames → neutral, sector-anchored names

Kept

  • Own ventures — VisLogic.AI, AMPPED, SASE-Experts
  • Public competitors, tools & market entities
  • Architecture, frameworks & method
  • Dollar figures & metrics (anonymized)
  • Real collaborators, credited by choice

The Level-Up

Quality control & validation cross-checks

This is the part worth memorializing. Sanitization without verification is just hope. Every file passed a battery of automated cross-checks before it was allowed to ship — and a few hard-won tooling lessons made those checks reliable.

Leak Sweep

Every page grep'd for client names, brands, and named individuals. The pass condition was absolute: a count of 0. False positives (like "compliance" matching "compli") were inspected, not assumed.

Link Integrity

The #1 failure mode. Every link in the index was resolved against the actual file list, character-for-character, before commit — no broken "Open" buttons, no orphaned destinations.

Parse Validation

Each file run through an HTML parser to confirm well-formedness — catching unclosed tags and structural breaks that a browser might silently paper over.

No Placeholders

A hard check for href="#" dead links. Coming-soon states were never allowed to ship as broken anchors.

Preservation Counts

Scrubbing is subtractive — so kept references were counted too. After the dashboard rebuild, all 46 platform-anchor references survived; the modeling stayed intact while the strategy was removed.

No Fabrication

When the numbers weren't in hand, we stopped and asked — rather than invent pilots or projections. The richest card waited for its real assets instead of guessing.

 Tooling lessons banked

  • Always name the file in every grep — a missing target silently hangs the command.
  • Normalize encoding (iconv) before sweeping files with emoji and smart-quotes.
  • Use commit dates to distinguish superseded assets from the day's work before deleting.
Durable Principles

What carries forward

Source hierarchy matters

Richer source files produce better cards. A master dashboard beats a thin brief every time.

Rebuild beats patch

When an artifact carried thematic contamination, rebuilding from the right source beat patching the wrong one.

Mine, don't just scrub

For pursuit-heavy docs, extracting the strategic concept beat scrubbing a hundred woven references.

Public vs. proprietary

Public competitors stay for credibility; proprietary names, execs, and req language go.

Path A discipline

Stage everything, commit once. One clean push beats a trail of iterative fixes.

Decisions as real choices

Surface the fork, explain the tradeoff, let the operator choose. Don't bury judgment calls.

Provable work, without exposed clients.

Fifteen engagements, distilled to method and architecture, verified end-to-end, and shipped as one clean release. The protocol is the asset now — repeatable on the next stack of artifacts.
